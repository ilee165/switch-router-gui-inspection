<objective>
Wire the session key through main.py: derive it at login, trigger the one-time
migration, propagate it to MainWindow, and from there to DeviceManagerDialog and
all panels. This completes the end-to-end key threading chain started in Plans
01, 03, and 04.

Purpose: Without this plan, the application will crash on startup because
MainWindow.__init__ now requires session_key, and connector calls will fail because
panels have session_key=None. This plan connects all the pieces.

Output: main.py with key derivation at login, migration trigger, and session_key
propagated to all five downstream consumers.
</objective>

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| login password (plaintext) → PBKDF2 → session_key | Plaintext password used exactly once for derivation; must not be stored after this point |
| session_key (in-memory) → MainWindow → panels/device_manager | Key flows via explicit constructor/method parameters; no global state |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03-M1 | Information Disclosure | login password used for PBKDF2 before being cleared | accept | pwd variable is local to _login; Python GC handles memory; no explicit zeroing (out of scope for desktop app threat model) |
| T-03-M2 | Information Disclosure | session_key stored on LoginDialog as self.session_key | accept | LoginDialog is destroyed after main() reads login.session_key and creates MainWindow; session_key lives on MainWindow._session_key thereafter |
| T-03-M3 | Information Disclosure | session_key passed as constructor argument to DeviceManagerDialog | accept | Dialog is modal; session_key is already in MainWindow memory; no new exposure surface |
| T-03-M4 | Denial of Service | migrate_plaintext_passwords fails during login | mitigate | If migration raises, it will propagate to _login and the error_lbl will show "Invalid credentials" — acceptable for edge-case DB corruption scenario. No data loss: unencrypted rows remain readable |
| T-03-SC | Tampering | No new packages installed in this plan | accept | No new installs; only modifications to existing files |
</threat_model>
