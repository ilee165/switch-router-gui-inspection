<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| device dict (ciphertext) → connector → ConnectHandler | Decryption happens here; plaintext exists only in the dict literal passed to ConnectHandler |
| panel → connector | session_key passes from UI layer to transport layer; must not be stored in UI state beyond self._session_key |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03-C1 | Information Disclosure | Plaintext password in connector dict | mitigate | Decryption happens inline in dict literal; no intermediate variable holds plaintext |
| T-03-C2 | Information Disclosure | session_key logged in exception | mitigate | _genie_fetch uses bare `except Exception: return None` — no logging of any parameter; pattern preserved as-is |
| T-03-C3 | Denial of Service | session_key is None when panel fetches before device selected | mitigate | BasePanel.fetch() guards `if not self._device: return` — session_key will only be None if device is also None, which is already blocked |
| T-03-SC | Tampering | No new packages installed in this plan | accept | No new installs; only modifications to existing files |
</threat_model>
