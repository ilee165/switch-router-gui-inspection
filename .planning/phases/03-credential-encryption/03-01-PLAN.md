    - derive_session_key(login_password: str, salt: bytes) -> bytes: uses hashlib.pbkdf2_hmac('sha256', login_password.encode('utf-8'), salt, 600_000) then base64.urlsafe_b64encode — returns Fernet-format bytes
    - get_or_create_salt(user_id: int) -> bytes: reads users.encryption_salt; if empty generates os.urandom(16), stores as salt.hex(), returns bytes
    - _is_fernet_token(value: str) -> bool: returns False if not value or not value.startswith('gAAAA'); then tries base64.urlsafe_b64decode(value + '==') and checks decoded length >= 57
    - migrate_plaintext_passwords(session_key: bytes) -> int: fetches all devices rows, for each checks _is_fernet_token on password and enable_pass, re-encrypts any that are plaintext, returns migrated count

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| login password → PBKDF2 → session_key | Plaintext password enters derivation; derived key must never leave memory |
| session_key → Fernet → SQLite | Ciphertext written to disk; key must not be written alongside it |
| migration loop | Reads plaintext from DB, writes ciphertext — plaintext only in memory during the loop iteration |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03-01 | Information Disclosure | SQLite devices table (password column) | mitigate | Fernet ciphertext stored; unreadable without login password + PBKDF2 derivation |
| T-03-02 | Information Disclosure | session_key in memory | mitigate | Key derived fresh each login, never written to disk; no global variable; verified by test_key_never_written_to_disk |
| T-03-03 | Elevation of Privilege | PBKDF2 brute-force | mitigate | 600,000 iterations — ~100ms per attempt on modern hardware; OWASP 2025 recommendation |
| T-03-04 | Denial of Service | Empty enable_pass raises InvalidToken | mitigate | Guard `if not value: return ''` in both encrypt_field and decrypt_field before any Fernet call |
| T-03-05 | Tampering | Double-encryption on device edit | mitigate | load_device() decrypts before form population (Plan 04); form always holds plaintext during edit |
| T-03-06 | Information Disclosure | Credential leaked in exception message | mitigate | decrypt_field does not catch and re-raise InvalidToken with the ciphertext in the message; exceptions propagate raw |
| T-03-SC | Tampering | cryptography package install | accept | Package is the canonical PyCA cryptography library (13 years, PyPI verified); legitimacy audit passed in RESEARCH.md |
</threat_model>
