<feature>
  <name>Credential encryption unit tests</name>
  <files>tests/conftest.py, tests/test_encryption.py</files>
  <behavior>
    Test 1 — test_save_device_stores_ciphertext (CRED-01):
      Call db.add_device with a known plaintext password and a valid session_key.
      Read the raw DB row. Assert the stored password starts with 'gAAAA', not equal to plaintext.

    Test 2 — test_key_derivation_deterministic (CRED-02):
      Call db.derive_session_key('hunter2', b'0123456789abcdef') twice.
      Assert both calls return equal bytes.

    Test 3 — test_key_never_written_to_disk (CRED-02):
      After deriving a session_key and calling add_device, open the DB file in binary
      mode and assert session_key bytes do not appear anywhere in the file content.

    Test 4 — test_migrate_plaintext_passwords (CRED-03):
      Insert a device row directly via SQL with a known plaintext password (bypassing
      add_device). Call db.migrate_plaintext_passwords(session_key). Read the row back.
      Assert the password now starts with 'gAAAA' and db.decrypt_field(session_key,
      stored_pw) == original plaintext.

    Test 5 — test_no_double_encryption (CRED-03):
      Call migrate_plaintext_passwords once on a plaintext row, then call it again.
      Assert the second call returns 0 (no rows re-migrated). Assert decrypt still
      returns the original plaintext (not double-encrypted garbage).

    Test 6 — test_get_device_returns_ciphertext (CRED-04):
      Add a device via add_device with a known plaintext password. Call db.get_device.
      Assert the returned dict's 'password' field starts with 'gAAAA' (ciphertext,
      not plaintext). Assert 'password' != the original plaintext.

    Test 7 — test_empty_enable_pass_safe (CRED-04):
      Add a device via add_device with enable_pass=''. Call db.get_device. Assert
      the returned enable_pass is '' (not a Fernet token). Assert
      db.decrypt_field(session_key, '') returns '' without raising.
  </behavior>
  <implementation>
    conftest.py: Use pytest's tmp_path fixture to create a temporary DB path.
    Monkeypatch db.DB_PATH to this temp path before each test. Call db.init_db()
    to create the schema in the temp location. This keeps tests fully isolated.

    test_encryption.py: Import db directly. Use a fixed test salt
    (b'testsalttestsalt', 16 bytes) and a fixed test password ('test_login_pass')
    to derive a consistent session_key across tests via db.derive_session_key.

    Use direct sqlite3 connection for the "insert plaintext" step in Test 4 — do
    NOT call add_device (which would encrypt). This simulates the pre-upgrade state.

    Install pytest if not present: pip install pytest
  </implementation>
</feature>
